# JobPilot AI Project Charter

## Project Vision

JobPilot AI is a cloud-native web application designed to help job seekers organize, track, and improve their job search.

The project is also a hands-on learning platform for mastering Amazon Web Services (AWS) while preparing for the AWS Certified Solutions Architect – Associate (SAA-C03) certification.

Rather than simply studying AWS concepts, every service learned will be applied to a real-world software project.

---

# Objectives

- Earn the AWS Certified Solutions Architect – Associate certification
- Build a professional cloud engineering portfolio
- Gain hands-on experience with AWS services
- Learn cloud architecture best practices
- Create a project suitable for technical interviews
- Document architectural decisions throughout development

---

# Target Users

Initially, the application is being built as a personal portfolio project.

Eventually, JobPilot AI may become a useful application for job seekers who want to:

- Track applications
- Upload resumes
- Store job descriptions
- Prepare for interviews
- Organize networking contacts
- Monitor application progress

---

# AI-Assisted Development

JobPilot AI will be developed using an AI-assisted engineering workflow.

Claude Code will be used to:

- Explore the codebase
- Propose implementation plans
- Assist with code generation
- Explain generated code
- Troubleshoot issues
- Refactor code
- Suggest tests
- Review implementation quality

AWS JobPilot Mentor will be used to:

- Guide AWS learning
- Plan project milestones
- Review architecture
- Explain cloud concepts
- Generate documentation
- Conduct certification quizzes
- Review completed work

All AI-generated code will be reviewed, understood, tested, and approved before being committed.

The developer remains responsible for all architectural decisions, code quality, testing, security, documentation, and final approval of every change.

# Guiding Principles

## Engineering Principles

- Learn by building
- Keep costs low
- Build incrementally
- Prioritize security
- Document everything
- Make architectural decisions intentionally
- Prefer simple, maintainable solutions over unnecessary complexity

## AI Development Principles

- AI assists engineering but does not replace engineering judgment.
- Every AI-generated change must be reviewed before acceptance.
- Understand every line of generated code before committing it.
- Prefer small, incremental AI-assisted changes over large code generation.
- Validate AI-generated work through testing and code review.
- Never commit code that cannot be explained.
- Treat AI as a collaborative engineering tool rather than an autonomous developer.
